#정보보안기사#보안

1. 현대 대칭키 암호

(1) 현대 블록 암호

1) 현대 블록 암호의 구성요소

(가) 개요

  1. 현대 블록 암호는 확산과 혼돈 성질 만족을 위해 전치 요소(P-박스), 치환 요소(S-box) 그 외 구성 요소 결합하여 설계됨.
  2. 공격 방지 암호화를 위해 이동요소(shift), 교환요소(swap), 분할요소(split), 조합요소, 전치장치(transposition, P-box), 치환장치(substitution, S-box), XOR연산의 조합으로 만들어짐.
  • 혼돈 : 암호문과 키의 상관관계를 숨김
  • 확산 : 평문의 통계적 성질을 암호문 전반에 퍼뜨려 숨김
  • 대칭키 암호 ≒ 관용 암호 ≒ 공통키 암호
  • 비대칭키 암호 ≒ 공개키 암호

(나) P-박스

  1. 문자 단위 암호화, 고전 장치 암호를 병렬적으로 수행
    블록 암호에서는 단순(straight) P-box, 확장(expansion) P-box, 축소(compresssion) P-box 세 가지 종류.
    => 아웃풋 크기가 동일, 확장, 축소됨.
  2. 축소 P-box
    n bit 입력 m bit 출력, n>m, 입력 비트 중 특정 비트 소실되 출력 안됨.
  3. 확장 P-box
    n bit 입력 m bit 출력, n<m, 입력 비트 중 특정 비트는 한 개 이상의 출력 비트로 연결.
    비트를 치환하고 동시에 다음 단계에서 비트의 양을 증가시키고자 할 때 사용.
  4. 역함수의 존재성
    단순 P-box는 역함수 존재
    축소 P-box, 확장 P-box 역함수 존재 안함

2) S-박스

(가) 개요

  1. S-box는 치환 암호의 축소. 입력과 출력 개수 달라도 됨.
  2. 역함수 존재성 : 존재할 수도 안할 수도 있다. 역함수 존재하는 S-box는 입력과 출력 비트 수 동일.

3) 합성 암호(Product Ciphers)

(가) 개요

  1. 치환, 전치, 그리고 그 밖의 구성요소를 결합한 복합적인 암호

(나) 확산(Diffusion)과 혼돈(Confusion)

  1. 합성 암호의 주된 개념은 설계된 블록 암호가 확산, 혼돈 성질을 갖도록 하는 것
  2. 확산 : 암호문과 평문 사이의 관계를 숨기는 것. 암호문 통계 테스트를 통해 평문을 찾는 것 방지
  3. 혼돈 : 암호문과 키의 관계를 숨김. 암호문을 이용해 키를 찾는 것을 방지. 즉, 키의 단일 비트가 변하면 암호문의 거의 모든 비트가 변함.

(다) 라운드(Rounds)

  1. 반복적 합성 암호를 통해 확산과 혼돈 얻음. 반복적 합성 암호를 라운드라고 칭함.

4) 두 가지 종류의 함성 암호

(가) Feistel 암호

  1. 3라운드 이상, 짝수 라운드로 구성. 라운드 함수와 관계없이 역변환 가능, 두 번의 수행으로 블록간의 완전한 확산이 이루어짐.
    알고리즘 수행 속도 빠르고, 하드웨어 및 소프트웨어 구현 용이, 아직 구조상 문제점 발견되지 않음.
    구조는 입력을 좌우 블록으로 분할하여 한 블록을 라운드 함수에 적용시킨 후의 출력 값을
    다른 블록에 적용하는 과정을 좌우블록에 대해 반복적으로 시행.
    라운드 키가 역순으로 작용한다는 점을 제외하면 암/복호화 과정이 동일하고 라운드 함수에 대한
    제약 조건이 없어 DES를 비롯한 대부분 블록암호에 채택.
  2. 암호 강도 결정 요소 : 평문 블록의 길이 >= 64 비트, 키 K의 길이 64 비트 내외, 라운드 수 >= 16회 권장
  3. 암호화, 복호화 과정 동일. 복호화 과정의 입력은 암호문과 보조키 KiK_{i}Ki​, 보조키의 입력 순서는 암호화 시 순서의 반대,
    즉, 복호화 첫 라운드 Kn−iK_{n-i}Kn−i​, 마지막 라운드 K1K_{1}K1​을 보조키로 입력.

(나) Feistel 암호 특징

  1. 입력 n 비트를 두 개의 블록 (L0,R0)(L_{0}, R_{0})(L0​,R0​)으로 나누어 라운드 함수 F, 라운드 키 KiK_{i}Ki​로 i 번째 라운드 과정
    주의) 최종 라운드에서 좌우 블록을 한 번 더 교환해야 함.

    Li=Ri−1L_{i} = R_{i-1}Li​=Ri−1​
    Ri=Li−1⊕Fi(Ri−1,Ki)R_{i} = L_{i-1} \oplus F_{i}(R_{i-1},K_{i})Ri​=Li−1​⊕Fi​(Ri−1​,Ki​)

  2. 페이스텔 네트워크는 많은 블록 암호 알고리즘에서 사용. AES 최종 후보 5개 중 3개 (MARS, RC6, Twofish)에서 사용. AES 최종 선정된 Rijndael은 사용 안함.
  • 대칭 블록 암호 구현
  • 블록 크기(Block size) : 블록 크기 크면 더 강한 보안, but 암 * 복호화 속도 저하. 128 비트가 조화로운 합리적 크기, 블록 암호 설계에서 공통적으로 이 크기 사용.
  • 키 길이(Key size) : 키 길이가 길다는 것은 더 강한 보안, but 암 * 복호화 속도 저하. 보편적 키 길이 128 비트.
  • 라운드 수 : 단일 과정으로는 보안 부족 => 라운드 수 증가시켜 보안 강화, 전형적 라운드 수 16.
  • 서브키 생성 알고리즘 : 복잡할수록 암호해독 어려움.
  • 라운드 함수 : 복잡할수록 암호해독 어려움.

(다) SPN 구조

  1. 블록 암호 라운드 함수 적용 방법에 따라 크게 두 가지 형태
    Substitution-permutation(SP) network, Feistel
  2. SP Network : 여러 개의 함수 중첩 시 개별 함수로 이루어진 암호보다 안전하다는 이론에 근거
    => 고전 암호 일종인 Substitution Cipher와 Permutation Cipher를 중첩하는 형태
  3. 입력을 여러 개의 소블록으로 나누고 각 소블록을 S-box로 입력하여 대치(Substitution)
    S-box의 출력을 P-box로 전치(permutation)하는 과정을 반복하는 방식

5) 블록 암호에 대한 공격

(가) 차분 분석(차분 해독법, Differential Cryptanalysis)

  1. Biham Shamir가 개발, 평문의 일부를 변경하면 암호문이 어떻게 변화하는지를 조사하는 암호 해독법
  2. 블록 암호는 입력 평문이 한 비트라도 달라지면 암호문은 전혀 다른 비트 패턴으로 변화 => 변화 형태 조사하여 해독

(나) 선형 분석(선형 해독법, Linear Cryptanalysis)

  1. Matsui가 개발, 평문과 암호문 비트를 몇 개 정도 XOR해서 0이 되는 확률을 조사
  2. 암호문이 충분히 랜덤하면 평문과 암호문의 비트를 몇 개 XOR한 결과가 0이 되는 확률은 1/2
    => 따라서 1/2 비율을 크게 벗어난 비트의 개수를 조사하여 키에 관한 정보 얻음
  3. 차분, 선형 해독법은 해독자가 임의로 만든 평문을 암호화할 수 있다는 가정 => 선택 평문 공격(Chosen Plaintext Attack)
    선형 해독법의 경우 근사 선형 관계성을 찾았다면 이를 통해 기지평문 공격에서 선택 평문 공격을 수행 가능

(다) 전수공격법(Exhaustive key search)

  1. Diffe와 Hellman이 제안한 방법, 모든 경우 전수 조사

(라) 통계적 분석(Statistical analysis)

  1. 암호문에 대해 평문의 각 단어별 빈도 등 통계적 자료 이용 해독

(마) 수학적 분석(Mathematical analysis)

  1. 통계적 방법을 포함, 수학적 이론 이용 해독

(2) 현대 스트림 암호

1) 개요

(가) 기본 개념

  1. 현대 스트림 암호에서 암호화와 복호화는 한번에 r 비트 생성.
    평문 비트 스트림 = P=pn⋯p2p1P=p_{n} \cdots p_{2}p_{1}P=pn​⋯p2​p1​, 암호문 비트 스트림 = C=cn⋯c2c1C=c_{n} \cdots c_{2}c_{1}C=cn​⋯c2​c1​,
    키 비트 스트림 = K=kn⋯k2k1K=k_{n} \cdots k_{2}k_{1}K=kn​⋯k2​k1​이라 할 때 (단, pi,ci,kip_{i},c_{i},k_{i}pi​,ci​,ki​는 r 비트 워드)
    암호화 : ci=E(ki,pi)c_{i} = E(k_{i},p_{i})ci​=E(ki​,pi​)
    복호화 : pi=D(ki,ci)p_{i} = D(k_{i},c_{i})pi​=D(ki​,ci​)
  2. 키 스트림 K=kn⋯k2k1K = k_{n} \cdots k_{2}k_{1}K=kn​⋯k2​k1​ 생성 방법이 현대 스트림 암호 주 관심사
    현대 스트림 암호 : 동기식, 비동기식 두 가지 종류로 분류

(나) 스트림 암호 설계 시 고려사항

  1. 암호화의 연속은 긴 주기를 가져야함
    의사 난수 생성기가 언젠가 반복되는 비트 스트림 생성하므로 반복 주기가 더 길어야 해독이 더 어려워짐
  2. 키 스트림은 진 난수 스트림과 최대한 비슷해야 함
    1, 0의 개수가 거의 동일해야 함, 키 스트림이 바이트 스트림으로 처리 시 256개의 가능한 바이트 값이 거의 같은 빈도로 출현해야 함
  3. 전사적 공격에 대응하기 위해서는 키가 충분히 길어야 함
    블록 암호에도 적용할 수 있기 때문에 현재 기술 수준에서 최소 128 비트인 것이 바람직

2) 동기식 스트림 암호

(가) 개요

  1. 동기식 스트림 암호에서 키 스트림은 평문 혹은 암호문 스트림과 독립적
  2. 키 스트림은 평문 혹은 암호문 비트와 키 사이에 어떠한 관계도 없이 생성, 사용

(나) One-Time Pad

  1. 동기식 스트림 암호 중 가장 간단, 안전
    암호화를 수행할 때마다 랜덤하게 선택된 키 스트림을 사용
  2. 해독 불가능하다는 것이 수학적으로 증명됨
    일회용 패드는 무조건 안전(unconditionally secure), 이론적으로 해독 불가능(theoretically unbreakable)로 알려짐
  3. 암호화 알고리즘, 복호화 알고리즘 각각 배타적 논리합 연산(XOR)을 사용. XOR의 성질로 암호화와 복호화 알고리즘은 서로 역관계
    XOR을 한 번에 한 비트씩 적용함
  • OTP 안전성을 위한 조건 : 패드는 한 번만 사용, 패드는 메시지 길이만큼 길어야함, 패드는 목적지로 안전하게 배포, 보호되어야 함, 패드는 순수하게 임의값으로 만들어져야 함
  • 현대 스트림 암호에서 평문 스트림의 r 비트 워드는 키 스트림의 r 비트 워드를 사용하여 대응되는 암호문의 r 비트 워드를 생성하여 암호화함
  • 동기식 스트림 암호에서 키는 평문 혹은 암호문과 독립적

(다) 귀환 시프트 레지스터(FSR, Feedback Shift Register)

  1. one-time pad의 절충안은 FSR이다. SW, HW 환경에서 모두 구현 가능, HW 구현이 더욱 용이함
  2. 시프트 레지스터와 귀환(feedback) 함수로 구성됨

(라) 선형 귀환 시프트 레지스터(LFSR, linear feedback shift register)

  1. HW로 쉽게 구현되며 많은 스트림 암호가 이용
  • LSFR의 최대 주기는 2m−12^m-12m−1이다.
  • 비동기식 스트림 암호에서 키는 평문 혹은 암호문에 종속적으로 결정됨
  • 현대 블록 암호는 모두 함성 암호이고 Feistel, non-Feistel 두 가지로 분류됨
    Feistel 암호는 역함수가 존재하는 요소와 존재하지 않는 요소 모두 이용 가능
    non-Feistel 암호는 오직 역함수가 존재하는 요소만 이용 가능

(마) 비선형 귀환 시프트 레지스터(NLFSR, Nonlinear Feedback Shift Register)

  1. LFSR이 선형성 공격에 취약, NLFSR로 더욱 안전한 스트림 암호 설계 가능
  • 강력한 스트림 암호 특징
    키스트림 값 내부에서 장기간 반복되는 패턴 없음
    통계적 예측 불가한 키스트림
    키스트림은 키와 선형적 관계에 있지 않음 => 키스트림값을 알아내도 키값을 아는 것이 아님
    통계적으로 치우치지 않은 키스트림(0과 1의 개수)

3) 비동기식 스트림 암호

  1. 각 스트림의 각 비트는 이전의 평문이나 암호문에 종속적으로 결정
  2. 블록 암호에서 다른 운영 모드를 생성하기 위해 사용되는 한 방법인 CFB(cipher feedback mode)는 실제로 스트림 암호를 생성

2. DES(Data Encryption Standard)

(1) 개요

1) 역사

  1. NIST 표준이고 IBM, NSA에서 개발
  2. NIST는 이제 3중 DES 사용 권고
  3. AES가 DES 대체하기 위해 새 표준으로 제정됨

2) 개관

  1. 평문 길이 64 bit, 키의 길이 56 bit, 평문이 64 bit보다 길면 블록으로 나눔
    Feistel 네트워크의 변형 형태, 16 라운드
    56 bit 원래 키로부터 16개의 서브키를 생성하고 이 것을 각 라운드에서 사용
  2. DES의 복호화 과정은 암호화 과정과 동일
    암호문을 알고리즘 입력으로, 서브키 KiK_{i}Ki​는 순서를 뒤집어서 적용.
    즉, 첫번째 라운드에 K16K_{16}K16​ 사용, 다음 라운드 K15K_{15}K15​, 마지막 16 라운드에 K1K_{1}K1​ 사용으로 반복
  • DES는 64 bit 평문을 64 bit 암호문으로 암호화하는 대칭키 암호화
  • DES의 키는 7 비트마다 오류검출 위한 정보가 1비트씩 들어가기 때문에 실질적으로는 56비트

(2) DES의 구조

1) 개요

  1. 암호화 과정은 두 개의 P-box와 16개의 Feistel 라운드 함수로 구성됨
    두 개의 P-box 중 하나는 초기 전치(initial permutation), 다른 하나는 최종 전치(final permutation)
  2. 각 라운드는 라운드 키 생성기에 의해 암호키로부터 생성된 48 bit 라운드 키 사용

2) 라운드 함수

(가) 개요

  1. DES는 16번 라운드 함수 사용. 각 라운드 함수는 Feistel 암호
  2. 라운드 함수는 이전 라운드 함수(또는 초기 P-box)의 출력 값 Li−1L_{i-1}Li−1​과 Ri−1R_{i-1}Ri−1​을 입력 받아, 다음 라운드(또는 최종 P-box)에 입력으로 전송될 LiL_{i}Li​와 RiR_{i}Ri​을 생성
  3. 각 라운드에는 혼합기(mixer)와 교환기(swapper) 존재, 이 것들은 역연산이 가능. 교환기는 명백히 역연산 가능

(나) DES 함수

  1. DES의 핵심은 DES 함수. 라운드 함수에 사용된 f(Ri−1,Ki)f(R_{i-1},K_{i})f(Ri−1​,Ki​)를 가리킴.
    DES 함수는 32비트 출력값을 산출하기 위해 가장 오른쪽 32비트 (Ri−1)(R_{i-1})(Ri−1​)에 48키를 적용
  2. DES 함수는 확장 P-box, 키 XOR, 8개의 S-box 그리고 단순 P-box의 4개 부분으로 구성됨
  • DES
    키 크기 : 56비트 + parity 8비트 = 64 비트
    안전성은 주로 비선형 함수로 구성된 8개의 S-box에 의존
    16라운드의 페이스텔 구조
    복호화는 암호화의 역순

3) 암호화 알고리즘과 복호화 알고리즘

(가) 개요

  1. mixer, swapper 사용해 16라운드 암호화, 복호화 알고리즘 만듦.
  2. 중요한 점은 암호화 키 순서의 역순으로 복호화 시 키 순서 적용

(3) DES 분석

1) 설계 기준

(가) S-box

  1. S-box는 각 라운드에서부터 그 다음 라운드까지 혼돈 만족하도록 설계됨
  2. S-box는 비선형 함수
  3. 입력값의 한 비트를 바꾼다면 출력값에서는 두 비트 이상이 바뀜

(나) P-box

  1. 32비트에서 32비트로 가는 하나의 단순 P-box와 32비트에서 48비트로 가는 하나의 확장 P-box 존재.
    두 개의 P-box는 비트들을 동시에 확산시킴.

2) DES의 취약점

  1. 평문 또는 키의 작은 변화가 암호문에 큰 변화를 만드는 쇄도효과(Avalanche Effect)가 매우 큼, 암호문의 각 비트가 평문의 많은 비트들에 의존하는 완비성(completeness)이 높은 것으로 증명되 암호문으로부터 평문 추론 매우 어려움
  2. 그러나 56비트 키 사용하므로 최대 키 공간의 크기 256(≅7.2×1016)2^{56}( \cong 7.2 \times 10^{16})256(≅7.2×1016)이 되고,
    개발 당시는 전사 공격(Brute Force Attack) 불가했지만, 현대에는 더 이상 안전하지 않음
  3. 실제로 전사 공격에 해독되고, 지금은 적용하면 위험

(4) 다중 DES

1) 3중 DES

(가) 개요

  1. 두 가지 버전 : 두 개의 키, 세 개의 키
  2. 3DES는 HW에서 효율적, SW 비효율적
    전자 여권 바이오 정보 보호, 금융 분야에서 활용

(나) 3중 DES 현황

  1. 3DES 은행 등에서 활용하지만 빠르지 않음
    과거 DES 암호화 자료와 호환성 중시하는 경우 제외하고 새 용도로 잘 안쓰임
  2. 3DES는 우리나라 표준은 아님, 우리나라 표준은 블록 암호 SEED, 국내 기관 공동 개발한 ARIA는 현재 사용됨
  • DES가 3DES보다 2.5배 정도 빠름, AES-128은 DES보다 3배 정도 빠름, RC4는 AES-128보다 2배 정도 빠름

(다) 두 개의 키를 갖는 3중 DES

  1. 두 개의 키 K1K_{1}K1​과 K2K_{2}K2​만 사용. 첫 번째와 세번째 단계에서 K1K_{1}K1​사용, 두번째 단계는 K2K_{2}K2​ 사용.
  2. 하나의 DES로 3중 DES를 만들기 위해 두 번째 DES는 복호화 알고리즘 사용, 1, 3 번째 경우 암호화 DES 알고리즘 사용

(라) 세 개의 키를 갖는 3중 DES

  1. 두 개 키를 쓰는 DES는 기지평문공격 가능성이 있어 세 개의 키를 사용하기도 함
  2. PGP와 같은 많은 응용프로그램에서 사용
  3. 3DES의 복호화는 암호화의 역이 됨 키3, 키2, 키1 순으로 복호화 -> 암호화 -> 복호화

(마) DES와의 호환성

  1. 트리플 DES에서 모든 키를 동일하게 하면 3DES는 보통의 DES와 같아진다. 1, 2 번째 DES가 암호화 후 복호화 하여 다시 평문이 나오는 꼴
  2. 과거 DES로 암호화된 암호문을 3DES 호환
구분(단위 bits) DES Triple DES AES
평문 블록 크기 64 64 128
암호문 블록 크기 64 64 128
키 크기 56 112 or 168 128, 192, or 256

3. AES

(1) 개요

1) 역사

  1. NIST에서 Rijndael을 AES로 선정, 벨기에 암호학자 둘이 개발

2) 선정 기준(Criteria)

  1. 안정성(security), 비용(cost), 효율성(implementation)

3) 라운드

  1. AES는 128비트 평문을 128비트 암호문으로 출력하는 알고리즘으로 non-Feistel
    10,12,14라운드를 사용, 각 라운드에 대응하는 키 크기 : 128, 192, 256 비트
  2. 키 크기에 따라 AES의 세가지 버전 존재 AES-(키 크기)
    그러나 어떤 버전이든 키 확장 알고리즘으로부터 생성되는 라운드 키 크기는 평문과 암호문 크기와 동일한 128 비트
  • SubBytes 연산은 16개의 독립된 바이트 단위의 변환 수행
  • SubBytes 변환과 InvSubBytes 변환은 서로 역변환 관계
  • Mixcolumns와 InvMixColumns는 역변환 관계

(2) 암호

1) 개요

  1. SPN(Substitution-Permutation Network) 구조
  2. AES 암호화 각 라운드는 비 선형성을 갖는 S-box를 적용하여 바이트 단위로 치환 수행하는 SubBytes() 연산,
    행 단위로 순환 시프트(Cyclic shift)를 수행하는 ShiftRows() 연산,
    높은 확산(diffusion)을 제공하기 위해 열 단위로 혼합(mixing)하는 Mixcolumns 연산,
    라운드 키와 state를 EX-OR하는 ADDRoundKey() 연산
  3. 암호화 마지막 라운드 MixColumns() 연산 수행 안함
  4. 알려진 블록 암호에 대한 공격에 안전하도록 설계됨, HW, SW 구현 시 속도나 코드 압축성 면에서 효율적이므로 스마트카드 등 응용에 적합
  • SPN 구조는 라운드 함수가 역변환이 되어야 한다는 제약이 있지만 더 많은 병렬성(parallelism)을 제공하기 때문에 암 * 복호화 알고리즘의 고속화를 요구하며, 컴퓨터 프로세서가 더 많은 병렬성을 지원하게 된 현 추세에 부응하는 방식
  • SPN은 입력을 여러 개의 소블록으로 나누고 각 소블록을 S-box에 입력하여 치환시키고 S-box의 출력을 P-box로 전치하는 과정을 반복
  • AES 4가지 암호화 과정(바이트 치환 변환, 행 이동, 열 혼합, 라운드 키 더하기) : 치환, 시프트, 그리고 DES에는 없는 비트 혼합
  • Feistel : DES, LOKI, CAST, Blowfish, MISTY, RC5, RC6, CAST256, E2, Twofish, Mars
  • SPN : SAFER, SHARK, Square, CRYPTION, Rijndael, SAFER+, Serpent, PRESENT

4. 기타 대칭키 암호 알고리즘

1) IDEA(International Data Encryption Algorithm)

  1. 스위스 연방 기술기관, 128 비트 키, 64 비트 블록 암호, Feistel과 SPN의 중간 형태인 Lai-Massey Scheme
  2. 8라운드에 걸쳐 데이터를 변환하고 마지막에 한 번 더 키를 적용시켜 64 비트 암호문을 생성 때문에 보통 8.5라운드라고 말함
    IDEA는 DES와 달리 S-box를 사용 안하고, 대수적 구조가 서로 다른 연상ㄴ을 교대 사용하여 암호 강도 증가
  3. 현재까지 특별한 분석 방법 알려지지 않아 DES에 비해 안전한 것으로 판단, PGP(Pretty Good Privacy)의 데이터 암호 알고리즘으로 채택 사용
    DES보다 2배 정도 빠르고, 무차별 공격에 더욱 효율적

2) RC5

  1. RSA 연구소의 Rivest가 개발, 비교적 간단한 연산으로 빠른 암호화, 복호화/ 모든 HW에 적합
  2. 입출력, 키, 라운드 수가 가변인 블록 알고리즘 RC5(Ron's Code 5)는 32/64/128 비트의 블록을 가지며, 속도는 DES의 약 10배

3) SEED

  1. 한국정보진흥원(현 한국인터넷진흥원)과 국내 암호전문가들이 함께 개발한 알고리즘/ 인터넷, 전자상거래, 무선 통신 등에서 공개될 경우 민감한 영향 끼칠 수 있는 중요 정보 및 개인 정보 보호 위한 대칭키 알고리즘
  2. 정보통신단체표준(TTA)으로 제정되고, 국제 표준화 기구 ISO/IEC 및 IETF에서 국제 블록암호 알고리즘 표준으로 제정됨
  3. 128 비트 비밀키에서 생성된 16개의 64 비트 라운드 키를 사용하여 총 16회의 라운드를 거쳐 128 비트의 평문 블록을 128 비트 암호문 블록으로 암호화하여 출력하는 방식
    256 비트 키를 사용하는 SEED-256으로 보다 강력한 암호화 제공
  4. SEED의 F 함수는 수정된 64 비트 Feistel 형태로 구성됨

4) ARIA(Academy Research Institute Agency)

  1. 국가보안기술연구소(NSRI) 주도로 힘을 모아 개발한 국가 암호화 알고리즘
  2. ARIA는 개발팀이 Academy(학계), Research Institute(연구소), Agency(정부기관)로 구성되었다는 의미, 학 * 연 * 관 공동 개발
  3. ISPN(Involutional SPN) 구조의 128 비트 블록 암호 128비트, 192비트, 256비트의 3종류 키 사용을 제공, 키의 길이에 따라 ARIA-128, ARIA-192, ARIA-256
  4. ARIA의 입출력 크기와 사용 가능한 키 크기는 미국 표준 블록암호인 AES와 동일

5) HIGHT(HIGh security and light weigHT)

  1. HIGHT는 RFID, USN 등과 같이 저전력 * 경량화를 요구하는 컴퓨팅 환경에서 기밀성을 제공하기 위해 KISA,ETRI 부설 연구소 및 고려대가 공동 개발한 64 비트 블록암호
  2. 정보통신단체표준(TTA)으로 제정됨, ISO/IEC 국제 표준 암호 표준으로 제정됨

6) LEA(Lightweight Encryption Algorithm)

  1. LEA는 국가보안기술연구소가 개발한 128 비트 경량 고속 블록 암호 알고리즘
    LEA는 현재 가장 많이 쓰이는 AES보다 1.5~2배 빠르다.
  2. 대용량 데이터를 빨리 처리하거나, 스마트폰 보안, 사물 인터넷(IoT) 등 저전력 암호화에 널리 쓸 수 있다.
  • 블록암호 요소 특징 : 블록 크기, 키 크기, 라운드 키 생성, 라운드 함수, 라운드 수

5. 현대 대칭키 암호를 이용한 암호화 기법

(1) 블록 암호의 사용 방식

1) 개요

(가) 기본개념

  1. 블록 암호는 고정 n비트 블록을 n비트 블록으로 변환시키는 함수
    임의 길이의 평문 암호화 위해 평문을 특정 길이로 분할, 블록 암호에 입력
    블록 암호 이용 평문 암호화 시 단순히 암호 알고리즘만이 아니라 사용 방식도 규정해야 함
  2. 블록 암호를 다양한 응용에 사용하기 위해 NIST에서는 5가지 운영 모드를 정의.
    이 5가지 모드는 사실상 블록 암호 사용 모든 암호 응용에 적용 가능
  3. 이 모드는 DES, AES 포함 어떤 대칭 블록 암호에도 적용 가능
    블록 암호 주요 모드는 다음과 같다
  • ECB : Electronic CodeBook mode(전자 부호표 모드)
  • CBC : Cipher Block Chaining mode(암호 블록 연쇄 모드)
  • CFB : Cipher-FeedBack mode(암호 피드백 모드)
  • OFB : Output-FeedBack mode(출력 피드백 모드)
  • CTR : CounTeR mode(카운터 모드)

2) Electronic Codebook 모드

(가) 개요

  1. 운영 모드 중에서 가장 간단한 모드는 ECB 모드.
    평문은 N개의 n비트 블록으로 분할됨.
  2. 만약 평문의 크기가 블록 크기의 배수가 아니라면, 평문의 마지막 블록에는 다른 블록들과 동일한 크기로 만들기 위하여 덧붙이기(충전물, padding) 필요
    각각의 블록을 암호화 또는 복호화하기 위해 사용되는 키는 동일

(나) 특징

  1. ECB 모드 장점은 블록 간의 독립성.
    다수 블록의 암호화와 복호화를 병렬적으로 수행 가능, 다수 블록 중에 일부 블록만 독립적으로 암호화 가능
    블록에서 발생하는 오류 다른 블록에 영향 없음
  2. ECB 방식은 평문의 블록 패턴과 암호문의 블록 패턴이 동일하게 유지되는 문제점 존재
    이런한 이유로 동일한 키로 여러 블록의 평문을 암호화할 때에는 ECB 방식 사용 권고하지 않으며
    보통 난수 발생과 같은 특수한 경우에 사용

(다) 응용

  1. ECB 모드의 장점은 매우 많은 데이터베이스를 암호화할 때 병렬적으로 처리 가능
  2. 데이터베이스에 암호화된 레코드 저장, 암호화된 레코드 복호화 해야 하는 경우 암호문 블록 독립성이 유용함
    다른 레코드에 영향을 주지 않고도 수정된 이후의 레코드를 중간에서부터 암호화 또는 복호화 가능

3) Cipher Block Chaining 모드

(가) 개요

  1. CBC 모드에서 각각의 평문 블록은 암호화되기 전에 이전 암호문 블록과 XOR됨.
    블록이 암호화될 때 암호화된 블록은 전송되지만 다음 블록을 암호화할 때 사용하기 위하여 메모리 저장됨.
  2. 첫 번째 블록 암호화 시 이전의 암호문 블록이 존재하지 않으므로,
    초기 벡터(Intial Vector)라는 허구의 블록 사용됨.
    송신자와 수신자가 사전에 공유한 IV를 존재하지 않는 C0C_{0}C0​ 대신에 사용

(나) 초기 벡터(Intial Vector)

  1. 주어진 평문에 대하여, IV 생성에 앞서 IV는 반드시 송수신 양자 모두가 알고 있어야 하고,
    제 3자의 예측 불가해야 함.
    평문과 연관지어 IV에 대한 예측 불가해야 함.

(다) 특징

  1. 초기치 또는 평문 P1,P2,…,PtP_{1}, P_{2}, \dots ,P_{t}P1​,P2​,…,Pt​에서 첫 번째 블록이 바뀌면 암호문 블록은 모두 바뀜.
    암호문 CiC_{i}Ci​에서 에러가 발생 시 평문 PiP_{i}Pi​와 Pi+1P_{i+1}Pi+1​에서 에러가 발생
    그러나 암호문 CiC_iCi​에서만 에러 발생 시 Ci+2C_{i+2}Ci+2​부터는 정상적으로 복호됨

(라) 암호화와 복호화

  1. CBC 모드에서는 1단계 전에 수행되어 결과로 출력된 암호문 블록에 평문 블록을 XOR하고 나서 암호화를 수행
    따라서 생성되는 각각의 암호문 블록은 현재 평문 블록뿐만 아니라 그 이전의 평문 블록들의 영향도 받게 됨(각 블록에 동일한 키 사용)
  2. 복호화 시 CBC 모드의 암호문 블록이 1개 파손된 경우, 암호문 블록의 길이가 바뀌지 않았다면 평문 블록에 미치는 영향은 2개 블록에 머뭄. 반면 평문 블록의 한 비트 오류는 모든 출력 암호문에 영향 미침.
  • CBC 모드에서는 1단계 전에 수행되어 결과로 출력된 암호문 블록에 평문 블록을 XOR하고 나서 암호화 수행
    C1=Ek(P1⊕IV)C_1 = E_k(P_1 \oplus IV)C1​=Ek​(P1​⊕IV)
    Ci=Ek(Pi⊕Ci−1),i=2,3,…,NC_i = E_k(P_i \oplus C_{i-1}), i = 2,3, \dots, NCi​=Ek​(Pi​⊕Ci−1​),i=2,3,…,N

(마) CBC 모드 활용

  1. 인터넷 보안 프로토콜 IPSec에서 통신 기밀성 위해 사용
  2. 3DES를 CBC 모드로 사용한 3DES-CBC나 AES를 CBC 모드로 사용한 AES-CBC 등이 여기에 해당
  3. CBC 모드는 인증을 수행하는 대칭키 암호 시스템의 하나인 Kerberos version 5에서도 사용됨

4) Cipher FeedBack 모드

(가) 개요

  1. CFB 모드 사용 시 어떤 블록 암호도 스트림 암호로 바꿀 수 있음
    스트림 암호의 경우 메시지 길이가 블록의 정수배가 되도록 패딩할 필요가 없고,
    실시간으로 사용 가능, 따라서 한 문자를 전송하는 경우 문자 중심 스트림 암호를 이용하여 각 문자 암호화되는 즉시 전송 가능

(나) 암호화와 복호화

  1. 복호화 과정은 평문 블록과 암호문 블록의 역할만 바뀌었을 뿐 동일
    암호화, 복호화 기법에서 암호 함수는 DES나 AES 사용(복호화 시 복호화 함수가 아닌 암호화 함수 사용)
  • CFB 모드와 같이 OFB, CTR 모드에서도 암호화와 복호화는 블록 암호의 암호화 함수 이용

(다) 스트림 암호로서의 CFB 모드

  1. 비록 CFB 모드가 DES와 AES 같은 블록 암호를 이용한 운영 모드지만, 그 결과는 스트림 암호와 같음.
    사실, 키 스트림이 암호문에 의존하는 비동기식(=자기 동기식=종속적) 스트림 암호

5) Output FeedBack 모드

(가) 개요

  1. OFB 모드는 평문 블록이 동일하면 암호문이 같아지는 ECB 모드의 단점과 오류 전파가 발생하는 CBC 모드와 CFB 모드를 개선한 동작 모드
  2. OFB 모드 동작은 암호기 출력과 평문을 EX-OR하여 암호문 생성하고 있으므로 오류 전파가 발생하지 않음
    그러나 암호문 송신자와 수신자 사이에 동기를 조절해야 함
    즉, 전송중인 암호문의 비트 손실이나 삽입 등에 유의해야 함
  3. OFB 모드에서도 CBC 모드나 CFB 모드와 마찬가지로 초기화 벡터(IV) 사용.
    초기화 벡터는 암호화할 때마다 다른 랜덤 비트열을 이용하는 것이 보통
  • CBC 방식은 평문 블록이 n비트로 되어 있어야 하기 때문에 r비트 단위로 암호화할 필요가 있는 특별한 경우엔 사용하기 어렵다.
    이러한 경우엔 CFB 방식(스트림 암호이기 때문)을 사용
  • OFB 방식은 에러가 확산되서는 안 되는 경우에 사용
    CFB 방식과 유사하게 r비트 단위로 암호화 가능
    CFB 방식에서는 암호문이 암호 알고리즘으로 다시 입력되는 것에 비해
    OFB 방식은 암호 알고리즘의 출력이 암호 알고리즘의 입력으로 사용됨

(나) 특징

  1. 초기치가 바뀌면 암호문은 모두 바뀜
  2. 암호 알고리즘의 출력은 평문과 무관
  3. 전송 중의 비트 오류 전파 안됨(C1C_1C1​에 비트 오류 발생 시 복원된 P1P_1P1​의 값에만 영향 미침, 이후 복원된 평문에는 아무런 영향 주지 않음)
  4. 암호문 CiC_iCi​에서 비트 손실이 발생하면 그 다음에 오는 평문은 모두 에러가 발생하기 때문에 동기를 새로 맞추어야 함

(다) 스트림 암호로서의 OFB 모드

  1. CFB 모드와 마찬가지로 OFB 모드는 블록 암호를 기반으로 한 스트림 암호이다.
    스트림은 평문이나 암호문과는 독립적이므로 동기식 스트림 암호이다.

6) CounTeR 모드

(가) 개요

  1. CTR 모드가 ATM(asynchronous transfer mode) 네트워크 보안과 IPSec(IP security)에 응용됨
  2. CTR 모드는 암호화 시 피드백 존재 안 함.
    하지만 키 스트림의 의사난수성은 카운터를 사용함으로써 성취될 수 있다.
  3. CTR 모드는 OFB 모드와 마찬가지로 이전 암호문 블록과 독립적인 키 스트림을 생성하지만 피드백을 사용하지 않는다.
    그리고 ECB 모드처럼 CTR 모드는 서로 독립적인 n비트 암호문 블록을 생성한다.

(나) 카운터 만드는 법

  1. 카운터의 초기값은 암호화 때마다 다른 값(비표)을 기초해서 만든다.

(다) 스트림 암호로서의 CTR 모드

  1. CFB, OFB처럼 CTR 모드는 실제로 스트림 암호(다른 평문 블록은 서로 다른 키로 XOR됨)
  2. OFB 경우와 같이 CTR 모드는 동기식 스트림 암호화 지원.
    CTR은 피드백이 없으므로 당연히 병렬처리 가능.
    CTR 모드는 OFB의 장점 대부분 지원
  • CFB, OFB 모드는 CBC, CTR 모드의 변형으로 봐도 무방하고, 현대에 주로 CBC, CTR 모드 사용함.

이름 장점 단점 비고
ECB - 간단
- 고속
- 병렬 처리 가능(암호화, 복호화 양쪽)		 - 평문 속의 반복이 암호문에 나타남
- 암호문 블록의 삭제나 교체에 의한 평문의 조작 가능
- 비트 단위의 에러가 있는 암호문을 복호화하면 대응하는 블록이 에러가 남
- 재전송 공격이 가능(Replay Attack)		 사용해서는 안 됨
CBC - 평문의 반복은 암호문에 반영 안 됨
- 복호화는 병렬 처리 가능
- 임의의 암호문 블록을 복호화 가능		 - 전송 도중 암호문 블록 CjC_jCj​에서 한 비트 오류가 발생하면 평문 블록 PjP_jPj​에는 대부분의 비트에서 오류 발생, 평문 블록 Pj+1P_{j+1}Pj+1​에서는 CjC_jCj​의 오류비트와 같은 위치에서 한 비트 오류 발생
- 암호화에서는 병렬 처리 안 됨
- Bit-flipping Attack 가능		 Practical Cryptography 권장
CFB - 패딩 필요 없음
- 복호화는 병렬 처리 가능
- 임의의 암호문 블록을 복호화 가능		 - 암호화에서는 병렬 처리 불가능
- 전송 도중 암호문 블록 CjC_jCj​에서 한 비트 오류가 발생하면 평문 블록 PjP_jPj​에는 CjC_jCj​의 오류 비트와 같은 위치에서 한 비트 오류 발생. 하지만 CjC_jCj​의 비트는 시픙트 레지스터에 오류가 존재하는 한 다음 평문 블록의 대부분의 비트에 오류 발생(약 50% 확률)
- 재전송 공격 가능
OFB - 패딩 필요 없음
- 암 * 복호화 사전 준비 가능
- 암 * 복호화가 같은 구조
- 비트 단위의 에러가 있는 암호문을 복호화하면 평문의 대응하는 비트만 에러가 남		 - 병렬 처리 불가능
- 적극적 공격자가 암호문 블록의 비트를 반전시키면 대응하는 평문 블록의 비트가 반전됨
- Bit-flipping Attack 가능
CTR - 패딩 필요 없음
- 암 * 복호화의 사전 준비 가능
- 암 * 복호화가 같은 구조
- 비트 단위의 에러가 있는 암호문 복호화 시 평문에 대응하는 비트만 에러 남
- 암호화, 복호화 모두 병렬 처리 가능		 - 적극적 공격자가 암호문 블록의 비트를 반전시키면 대응하는 평문 블록의 비트가 반전 됨
- Bit-flipping Attack 가능		 Practical Cryptography 권장
